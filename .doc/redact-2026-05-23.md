## Design Decisions

**`save_new()` copies all xref IDs** — The simplest correct approach. Object IDs are unchanged; only byte offsets differ in the new file. Avoids ID remapping complexity and the risk of broken cross-references. The `log::warn!` on skipped objects provides a safety net for corrupt source documents.

**Per-span text dropping (not per-glyph)** — Precise per-glyph width computation requires the font cache (gated behind the `render` feature). Conservative span-level dropping is safer for redaction: it may remove slightly more text than strictly necessary, but it guarantees nothing leaks through a partial-drop.

**Conservative `Do`/`BI` handling** — Form XObject recursion requires a full content-stream interpreter. Dropped if the CTM-mapped unit square intersects a zone. False positives (dropping non-intersecting XObjects) are accepted in v1; noted as follow-up.

**Clipping paths (`W`/`W*`) never dropped** — Dropping a clipping operator corrupts the graphics state for all subsequent content; they are always flushed through.

**No new dependencies** — The redact module uses only crate-internal types (`Matrix`, `PdfObject`, `ContentBuilder`, `parse_content_stream`, `serialize_operations`, `make_flate_stream`). WASM build is unaffected.

**Overlay always appended** — Even if the rewriter drops no content, the overlay rectangle is appended. This ensures the visual result is consistent regardless of whether content was detected inside the zone.

**`AnnotationType::Redact` with `/IC`** — Follows ISO 32000-2 §12.5.6.23. The `IC` entry (interior colour) is the standard field for the fill colour shown after redaction is applied.

## Known Limitations / Follow-up

- **Form XObject recursion**: `Do` operators are dropped conservatively (CTM unit-square test). A future phase should recurse into form streams and apply the rewriter recursively.
- **Per-glyph text precision**: Text bbox uses a `0.6 * font_size * char_count` width estimate. For precise character-level splitting (e.g. redact only part of a word), font width metrics from the font cache are needed.
- **Inline image detection**: `BI` is treated the same as `Do` (unit-square test). A more accurate approach would use the image's `/Width` and `/Height` together with the CTM.
- **Object cleanup**: Unreferenced objects from the original document survive in `save_new()` output. A future garbage-collection pass could trim them.
- **AES-256 encrypted PDFs**: Redaction requires parsing, which requires decryption. R5/R6 (AES-256) are not yet supported by the `crypto` feature.
