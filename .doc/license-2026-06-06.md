## Design Decisions

### HMAC-SHA256 over a custom binary format
The key is a 105-byte payload (tier + expiry + licensee + signature) encoded as base64url. No external license server, no network dependency, and safe for WASM where there may be no TLS stack.

### `OnceLock<License>` for activation state
Thread-safe single-assignment. Once activated, the tier is immutable for the process lifetime — no race conditions and no TOCTOU window between checking and using the tier. The downside is keys cannot be "rotated" without restarting, which is acceptable for a WASM session model where the page load is the natural boundary.

### `#[cfg(feature = "crypto")]` gate on the entire module
The license module and all enforcement calls are absent when `crypto` is not enabled. This means developer builds without crypto can iterate freely; enforcement is a production-only concern. The `wasm` feature was updated to imply `crypto` so WASM always has enforcement.

### `#[cfg(feature = "writer")]` gate on `watermark` submodule
The watermark writes to the PDF via `ContentBuilder` and `PdfEditor`, which require the `writer` feature. Splitting the submodule gate avoids pulling in writer code when only doing license validation.

### `#[cfg(test)]` bypass in `require()`
`OnceLock` cannot be reset between unit tests in the same process. A per-test tier requires either `AtomicU8` (architectural change) or thread-locals. The chosen approach bypasses enforcement in test builds and instead tests the enforcement logic directly (tier comparison) in `license::tests`. Integration tests use `ensure_pro_license()` which activates Pro once at process start.

### `encode_license_key` is `pub`
Made public so integration tests and the `keygen` binary can share the same key-generation code. In a production deployment the secret (`LICENSE_SECRET`) would be injected via `build.rs` from an environment variable, making the test keys valid only for builds that embed the test secret.

## Test Coverage

### `src/license/mod.rs` (unit tests)
- `valid_pro_key_decodes` — happy-path: Pro tier, perpetual, correct licensee
- `valid_enterprise_key_decodes` — happy-path: Enterprise tier
- `free_key_decodes` — happy-path: Free tier

## Known Limitations / Follow-up

- **`extract_pages` not gated** — the function listed in the phase spec does not yet exist in `src/editor/merge.rs`. Gate will be added when the function is implemented.
- **`LICENSE_SECRET` is a placeholder** — must be replaced with a value from `std::env!("PDF_CORE_LICENSE_SECRET")` in `build.rs` before any public release. The current placeholder means anyone who reads the source can forge keys.
- **WASM expiry not enforced** — `SystemTime` is unavailable in WASM. A future improvement could accept a `current_timestamp_ms` parameter from JS (e.g. `Date.now()`) to enable expiry checks in the browser.
- **Enterprise tier has no gated functions yet** — the three-tier model is in place but only Pro gates are wired. Enterprise-specific features (signatures, PDF/A, REST API) will add `Tier::Enterprise` gates as those modules are built.
- **`require()` is bypassed in test builds** — enforcement tests rely on direct tier comparisons rather than end-to-end calls through gated functions. This is safe for now but means a test that accidentally calls a gated function with a Free tier will not catch the regression.
